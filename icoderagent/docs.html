Documentation

A real AI coding agent for iPhone, iPad and Mac. Bring your own model key; your code and keys stay on your device.

Overview

An autonomous agent that plans, writes files, runs code and iterates until your task is done — on-device.

Autonomous
Give it a task in plain language; it works the whole loop, not a single reply.
Embedded Python
A full CPython runtime is built in, so it can run the code it writes.
Bring your own key
Uses your own provider account; keys and code never touch our servers.

Install

iPhone & iPad
Install from the App Store. Runs on-device; a free tier lets you start.
Mac companion
Install the desktop companion (download) to run real projects on a computer.

First agent run

1 · Pick a provider
Settings → AI Provider → choose one and paste your API key.
2 · Give a task
In Terminal, type e.g. agent build a tip calculator page.
3 · See the result
Open the Preview tab to view web output it builds.

Bring your own key

Model calls are billed by your provider directly to you; we never resell AI usage.

Keys are stored in the iOS Keychain and sent only to the provider you chose.

Supported providers

Cloud
Anthropic (Claude), OpenAI (GPT), xAI (Grok).
More cloud
Gemini, GitHub Models, Groq, Azure, OpenRouter, DeepSeek, Moonshot, Hugging Face.
Local
Ollama and Apple MLX servers you run yourself.
On-device
Edge models that run entirely on your device — no key, no network.

Choosing a model

Pick from each provider's list, or type a custom model ID. Stronger models code better; smaller ones are fine for quick edits.

How it works

Each turn the agent calls tools — write, read, run, search, screenshot — reads the result, and decides the next step, committing checkpoints as it goes.

Plan & Loop

plan <task>
Lays out an approach before writing code — good for larger changes.
loop <goal>
Runs a goal to completion on its own: build, verify, iterate until done.

Sub-agents

The agent can delegate subtasks, run work in parallel, or run a plan → build → review pipeline. It does this automatically when it helps.

Token saver

Compacts noisy tool output on-device before it reaches the model, so you spend fewer tokens. On by default.

On-device models

Run small models entirely on your device — no key, no network, no per-token cost, and they don't count against your quota.

Download under Settings → On-Device Models. Larger models need a recent device with more memory.

Ollama & MLX

Ollama
Choose Ollama and enter your computer's LAN URL (e.g. http://192.168.1.5:11434).
Apple MLX
Run mlx_lm.server on an Apple-silicon Mac and enter its URL (e.g. :8080).

Use your computer's LAN IP, not localhost — the phone reaches it over your network.

Desktop companion

A small app for macOS, Windows and Linux that runs real shell tools (compilers, npm, pytest, Docker) on your computer, driven from your phone over a secure relay.

Connect a computer

1 · Open the companion
Sign in with the same account and pick a project folder.
2 · Select it in the app
Settings → Remote → choose your computer (it appears automatically).

No tunnels or port forwarding — the companion connects out to the relay itself.

Remote mode

With a computer connected, files sync both ways and commands run on the computer's full toolchain. Without one, the agent uses the on-device Python only.

SSH hosts

Register extra machines (Settings → Remote Hosts) and have the agent run commands on them by alias with ssh_run.

Skills

Reusable Markdown playbooks the agent loads on demand — a procedure for something you do often. Manage under Settings → CognitiveSkills.

MCP servers

Add tools via the Model Context Protocol (GitHub, Linear, docs, and more) under Settings → Connections (MCP).

Remote (HTTP)
Works directly on your device.
Local (stdio)
Runs on your connected computer via the companion.
Browse community
Search the public registry and add one — you review it first.

Caution: community servers aren't vetted; a local server runs code on your computer. Only add publishers you trust.

Self-authored tools

When it hits a repeated need, the agent can write a new tool — but shows you the exact code and waits for your approval before it runs.

Projects & Files

The agent works in a project folder — on-device, or mapped to your computer's folder with a companion connected. Edit files in the Files tab.

Preview & Design

Preview renders HTML the agent builds; screenshots also report any runtime error, so a blank screen comes with the reason. Design is a light sketching surface.

Databases & Git

Run SQL on saved connections and use Git for version control. New projects are scaffolded with a .gitignore, .env and README.md, and the agent auto-commits checkpoints.

Plans & billing

Subscriptions cover app usage (a monthly quota of conversations), not the AI — you pay your provider for model use.

Free
10 conversations, no history retained.
Starter
500 / month · 100 chats of history.
Growth
1,550 / month · 300 chats of history.
Scale
5,000 / month · 1,000 chats of history.

Unused conversations roll over while subscribed. On-device model use doesn't count. Manage in your Apple subscription settings.

Troubleshooting

"No computer is connected"
Shell tools need the companion — connect it under Settings → Remote.
Hit the paywall
Subscribe, or switch to an on-device model (doesn't count against quota).
Local model URL not set
Enter your computer's LAN IP for Ollama/MLX, not localhost.
Still stuck
See Support or email support@aidigiapps.com.

Privacy

Keys live in the Keychain and go only to your provider. Projects, history and logs stay on-device. No ads, no third-party analytics, no tracking. See the privacy policy.
